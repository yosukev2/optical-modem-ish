# Risks & Mitigations

## 目的
詰みポイント（リスク）を早期に言語化し、回避策を決めておく。

---

## R1: 受光側が不安定（ノイズ/浮き/配線長）
- 症状：Rxが揺れる、同期が落ちる、CRC failが多発
- 回避策：
  - GNDを短く、配線を短く
  - 入力プル抵抗検討（必要なら）
  - ロジアナ/オシロで波形確認を“必須工程”にする
  - まず低速（0.5Mbps）で安定化してから上げる

## R2: フットプリント/配線ミスで基板が詰む
- 症状：Tx/Rxが動かない、電源が出ない
- 回避策：
  - データシート照合（フットプリント最優先）
  - テストパッド（5V/3.3V/GND/Tx_IN/Rx_OUT）を必ず入れる
  - DRC/ERCゼロ、発注前セルフレビューを docs に残す

## R3: “動くけど評価できない”状態になる（ログ/テスト不足）
- 症状：成功/失敗の比較ができない、再現できない
- 回避策：
  - W1でログ形式とテスト計画を先に置く
  - BER/PERを自動化してCSVに残す
  - 不具合は必ずチケット化し、再発防止としてテスト追加する
